# 【初赛 Demo】AI 周末规划师 —— 让 AI 为你规划完美的周末出行

> Trae AI 创意大赛初赛作品 · 在线体验：https://weekend-planning-rouge.vercel.app/

## 一、创意介绍

"周末去哪玩？"——这大概是每周五下午被问得最多的问题。查小红书、翻大众点评、比高德路线、算预算、凑朋友时间……一份像样的周末计划，往往要花掉大半个晚上。**AI 周末规划师**想做的，就是把这套繁琐流程压缩到 3 分钟：你说一句"杭州、明天、2 个人、500 块、想放松吃美食"，AI 就把完整行程、地图路线、时间轴、天气、花费清单一次性端到你面前。

产品形态是一个纯前端 + Serverless 的 Web 应用，无需注册登录，打开即用。它不是"生成一段 Markdown 文案"的玩具，而是一套**端到端的行程工程**：AI 流式生成结构化 JSON → 后端用高德 POI 反查真实坐标覆盖 AI 编造的坐标 → 地理最近邻算法重排路线减少折返 → 地图绘制 + 时间轴联动 + 多人协同收集偏好。每一个环节都经得起工程审视。

更特别的是，这个项目全程在 **Trae IDE 的 agent 蜂群模式**下开发：多轮并行 agent 协作完成了地图优化、JWT 修复、性能分包、模型切换等关键工程任务——AI 不只是写代码的助手，更是一个可以并行调度、分工协作的"研发团队"。

## 二、目标用户与痛点

- **忙碌的上班族**：周末想出去玩，但没时间做攻略，希望 3 分钟拿到可执行方案。
- **情侣 / 闺蜜 / 小家庭**：多人出行需要凑齐偏好，协调成本高，需要"收集偏好 → 融合重生成"的协同闭环。
- **异地短途游用户**：对目的地不熟，需要真实坐标、地图路线和天气提示，而非 AI 一拍脑袋编的经纬度。
- **决策困难症**：面对海量景点信息选不出来，需要 AI 主动给方案、给理由、给预算对照。

痛点一句话概括：**攻略散落在十几个 App 里，AI 生成的内容又不一定靠谱，规划周末比过周末还累。**

## 三、核心功能演示

- **AI 流式生成 + 4 阶段状态机**：顶部阶段轨道「理解需求 → 生成方案 → 优化细节 → 完成」实时反映生成进度，AI 思考过程逐字流式输出，坐标补充进度条可见——不是假进度条，阶段切换由 reasoning/content 长度真实驱动。
  [截图1：AI 流式生成 4 阶段状态机，阶段 2 激活、连接线流动光效、思考过程逐字输出]

- **高德地图路线 + 地理最近邻重排**：5 个景点在地图上自动连线，支持「按地理 / 按时间」一键切换；"按地理"模式用最近邻算法重排点位减少折返，地图组件懒加载，首屏 bundle 从 128KB 降到 6KB。
  [截图2：高德地图绘制当日路线，5 个标记点连线、右上角排序切换器]

- **多人协同 + Supabase Realtime**：一键生成分享链接，小伙伴匿名加入并提交偏好，偏好通过 Realtime 实时推送给创建者，创建者可"基于全员偏好重新生成"行程，实现真正的协同规划闭环。

- **时间轴 + 地图双向联动**：纵向渐变时间轴串联全天行程卡片，点击地图标记自动滚动定位并高亮对应卡片，GSAP stagger 入场与节点 pulse 动效。

- **天气智能提示 + 总花费数字滚动**：高德天气 API 返回实时天气与出行建议，总花费从 0 弹性滚动到目标值，预算对照一目了然。

- **演示兜底 + 超时保护**：AI 偶尔抽风也不怕，50s 后端硬超时 + 30s 前端友好提示 + 一键加载真实坐标的演示行程，评委体验零阻断。

## 四、技术亮点

| 亮点 | 实现 | 价值 |
| --- | --- | --- |
| **SSE 流式** | fetch + ReadableStream，自研 `parseSSEBuffer` 拼接不完整片段，事件流 `meta/reasoning/chunk/progress/done/error` 分发 | AI 推理与生成逐字呈现，体验真实 |
| **4 阶段状态机** | `stages` computed 由 reasoning/content 长度驱动切换，替代假进度条 | 进度反映真实生成状态，不是骗人的转圈 |
| **vendor 分包** | `vite.config.ts` manualChunks 拆分 vue/supabase/gsap/lucide，地图组件 `defineAsyncComponent` 懒加载 | 首屏 chunk 128KB → 6KB，95% 体积优化 |
| **JWT 修复** | `optionalAuth` 中间件用 `supabaseAdmin.auth.getUser(token)` 校验匿名会话，失败不阻断 | 匿名协同安全可信，分享/加入链路稳定 |
| **50s 超时保护** | 后端 `TOTAL_BUDGET_MS = 50_000`（留 10s 给 enrichment + done），AI 客户端 55s 超时，避开 Vercel 60s 函数上限 | 不踩 Serverless 超时红线 |
| **高德双 Key 体系** | Web 服务 Key（后端 POI/天气/geocode）+ JS API Key（前端地图），8s 单请求超时兜底 | 前后端职责分离，Key 安全不外泄 |
| **坐标赋分反查** | 距离分 40% + 名称匹配分 40% + 类型匹配分 20%，全并发 POI 搜索，geocode 兜底 | 用真实坐标覆盖 AI 编造的经纬度 |
| **演示兜底** | 生成失败/超时一键加载真实坐标的演示行程 | 评委体验零阻断，容错友好 |

## 五、TRAE 实践过程（参赛重点）

本项目全程在 **Trae IDE 的 agent 蜂群模式**下开发。区别于"一个 agent 串行写完所有代码"的传统模式，蜂群模式允许**并行调度多个 agent 分工协作**：诊断 agent 负责定位问题、修复 agent 负责改代码、验证 agent 负责跑构建，多个 agent 同时跑、互不阻塞，把原本需要数小时的"排查—修复—验证"串行循环压缩到分钟级。这是本项目在开发模式上最大的创新点。

## 六、价值与意义

对用户而言，AI 周末规划师把"规划周末"从一件耗时耗力的苦差事，变成 3 分钟的轻松体验，让更多人愿意、也敢于在周末走出去。

对周末经济而言，它降低了出行的决策门槛：更精准的路线、更可控的预算、更多人的协同参与，意味着更多本地消费被激活——每一份 AI 生成的行程，都可能是一笔真实的周末消费。

对 AI 应用范式而言，它证明了"AI 生成 + 工程兜底"的可行性：不盲信 AI 输出的坐标，用高德 POI 反查覆盖；不赌 AI 一定在线，用演示兜底保体验；不让 AI 单打独斗，用 agent 蜂群并行开发。这是 AI 应用从"Demo 玩具"走向"可工程化产品"的一个真实样本。

## 七、在线体验

🌐 体验地址：https://weekend-planning-rouge.vercel.app/

操作引导（3 步即可）：

1. **打开首页 → 点击「一键填充示例」**：表单自动填入杭州示例（城市/日期/预算/人数/偏好），3 秒进入生成环节。
2. **点击「生成周末行程」**：观察 4 阶段状态机与 SSE 流式输出，约 10–30 秒拿到完整行程（地图路线 + 时间轴 + 天气 + 清单）。
3. **点击「🔗 分享链接」**：用无痕窗口打开链接加入协同，提交偏好后创建者可基于全员偏好重新生成——体验多人协同闭环。

> 兜底承诺：若 AI 生成失败或超时，随时点击「🎯 加载演示行程」立即体验完整杭州一日游效果，演示零阻断。
