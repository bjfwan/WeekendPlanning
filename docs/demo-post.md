# 【生活娱乐】AI 周末规划师 —— 让 AI 为你规划完美的周末出行

> Trae AI 创意大赛参赛作品 · 生活娱乐赛道

## 一、Demo 简介

**AI 周末规划师**是一个纯前端 + Serverless 的 Web 应用——你说一句"杭州、明天、2 个人、500 块、想放松吃美食"，AI 就在 3 分钟内把完整行程、地图路线、时间轴、天气和花费清单一次性端到你面前。无需注册登录，打开即用。

**面向谁**：

- **忙碌的上班族**：周末想出去玩，但没时间做攻略；
- **情侣 / 闺蜜 / 小家庭**：多人出行需要凑齐偏好，协调成本高；
- **异地短途游用户**：对目的地不熟，需要真实坐标和地图路线；
- **决策困难症**：面对海量景点信息选不出来，需要 AI 主动给方案、给理由、给预算对照。

**核心功能**：

1. **AI 流式生成行程 + 4 阶段状态机**
   顶部阶段轨道「理解需求 → 生成方案 → 优化细节 → 完成」实时反映生成进度，AI 思考过程逐字流式输出，坐标补充进度条可见——不是假进度条，阶段切换由 reasoning/content 长度真实驱动。
   [截图：03-generating.png —— AI 流式生成 4 阶段状态机，阶段 2 激活、连接线流动光效、思考过程逐字输出]

2. **高德地图多交通路线绘制 + 地理最近邻重排**
   5 个景点在地图上自动连线，支持「按地理 / 按时间」一键切换；"按地理"模式用最近邻算法重排点位减少折返。地图组件懒加载，首屏 bundle 从 128KB 降到 6KB。
   [截图：04-map-route.png —— 高德地图绘制当日路线，5 个标记点连线、右上角排序切换器]

3. **多人协同 + Supabase Realtime**
   一键生成分享链接，小伙伴匿名加入并提交偏好，偏好通过 Realtime 实时推送给创建者，创建者可"基于全员偏好重新生成"行程，实现真正的协同规划闭环。
   [截图：06-share-modal-real.png —— 分享弹窗，生成分享链接邀请好友匿名协同]

## 二、Demo 创作思路

**灵感来源**：每周五下午，"周末去哪玩？"大概是打工人被问得最多的问题。查小红书、翻大众点评、比高德路线、算预算、凑朋友时间……一份像样的周末计划，往往要花掉大半个晚上。

**想解决的问题**：攻略散落在十几个 App 里，AI 生成的内容又不一定靠谱——经纬度可能是一拍脑袋编的、路线可能折返跑、预算可能对不上。一句话痛点：**规划周末比过周末还累**。

**为什么做这个方向**：

- **高频刚需**：每周都有需求，覆盖人群广，不是一次性工具；
- **可工程化**：AI 生成 + 工程兜底能真正落地——用高德 POI 反查覆盖 AI 编造的坐标，用最近邻算法重排路线，用超时保护和演示兜底保体验；
- **有协同价值**：多人出行天然需要"收集偏好 → 融合重生成"的闭环，这是单机 AI 工具做不到的差异化能力。

它不是"生成一段 Markdown 文案"的玩具，而是一套端到端的行程工程：AI 流式生成结构化 JSON → 后端用高德 POI 反查真实坐标覆盖 AI 编造的坐标 → 地理最近邻算法重排路线减少折返 → 地图绘制 + 时间轴联动 + 多人协同收集偏好。每一个环节都经得起工程审视。

## 四、TRAE 实践过程

本项目全程在 **Trae IDE 的 agent 蜂群模式**下开发。区别于"一个 agent 串行写完所有代码"的传统模式，蜂群模式允许**并行调度多个 agent 分工协作**：诊断 agent 负责定位问题、修复 agent 负责改代码、验证 agent 负责跑构建，多个 agent 同时跑、互不阻塞，把原本需要数小时的"排查—修复—验证"串行循环压缩到分钟级。

这是本项目在开发模式上最大的创新点——**用 AI 协作开发 AI 产品**：Trae IDE 不只是代码补全工具，更是一个可以分工调度的 AI 研发团队，对独立开发者和小团队来说是真正提效的范式转变。

**关键工程决策（均在 TRAE 中由 agent 协作完成）**：

- **SSE 流式自研**：fetch + ReadableStream，自研 `parseSSEBuffer` 拼接不完整片段，事件流 `meta/reasoning/chunk/progress/done/error` 分发，AI 推理与生成逐字呈现；
- **4 阶段状态机**：`stages` computed 由 reasoning/content 长度驱动切换，替代假进度条，进度反映真实生成状态；
- **vendor 分包**：`vite.config.ts` manualChunks 拆分 vue/supabase/gsap/lucide，地图组件 `defineAsyncComponent` 懒加载，首屏 chunk 128KB → 6KB，95% 体积优化；
- **JWT 修复**：`optionalAuth` 中间件用 `supabaseAdmin.auth.getUser(token)` 校验匿名会话，失败不阻断，匿名协同安全可信；
- **50s 超时保护**：后端 `TOTAL_BUDGET_MS = 50_000`（留 10s 给 enrichment + done），AI 客户端 55s 超时，避开 Vercel 60s 函数上限；
- **高德双 Key 体系**：Web 服务 Key（后端 POI/天气/geocode）+ JS API Key（前端地图），8s 单请求超时兜底，前后端职责分离，Key 安全不外泄；
- **坐标赋分反查**：距离分 40% + 名称匹配分 40% + 类型匹配分 20%，全并发 POI 搜索，geocode 兜底，用真实坐标覆盖 AI 编造的经纬度。

## 五、后续规划：从单轮生成到多轮 Agent 协作

当前版本虽已跑通"一句话生成完整行程"的闭环，但我们清楚它的天花板——**这是一次性生成**：用户填完所有信息 → AI 一口气输出整段行程，中间无法主动查资料、无法动态调整策略，更像"一问一答"而非真正的协作。我们不想止步于此。

**演进方向：分步填写 + 多轮 Agent（像 Coding 一样规划行程）**

把行程规划从"单轮 prompt"升级为"像 Coding 一样的多步 Agent 流"：用户每填写一个关键信息（城市 / 日期 / 预算 / 偏好），就触发一个独立的 agent 任务。每个 agent 可以：

- **主动搜索实时信息**——目的地天气、景点开放状态、交通管制、临时活动；
- **接入 MCP 工具**——高德 POI 查询、票务 API、天气 API，让数据从"AI 猜"变成"工具取"；
- **调用 TRAE Skill**——注入旅行领域专业知识库，避免通用模型的常识盲区；
- **与用户交互确认**——"检测到周末有音乐节，是否调整路线避开拥堵？"关键决策点回到人类。

多个 agent 串行 / 并行协作，像 coding 一样逐步构建、迭代行程，参考 OpenAI GPTs、AutoGPT、MetaGPT 等开源 agent 设计模式。

**技术构想**

- **Agent 状态机**：规划 → 搜索 → 生成 → 校验 → 优化，每一步可回溯、可分支；
- **工具链**：MCP 接入高德 POI / 天气 / 票务，Skill 接入旅行领域知识；
- **Human-in-the-loop**：关键节点让用户拍板，AI 不替用户做价值判断；
- **行程版本管理**：每轮 agent 产出可对比、可回滚的行程版本，像 Git 一样管理"方案演进史"；
- **个性化记忆系统**：让 agent 从"通用出方案"进化为"懂你的私人规划师"——
  - **用户喜好文档**：用户可维护个人偏好文档（如"喜欢自然风光，避开商业景点""预算敏感，优先免费景点""不吃辣"等），agent 生成行程时自动参考，省去每次重复输入；
  - **历史偏好学习**：基于用户历史行程数据自动沉淀偏好画像（常去景点类型、消费习惯、出行时间偏好等），越用越懂你；
  - **规则引擎**：用户可定义硬性规则（如"周日晚上不安排远距离景点""预算超 80% 时提醒"），agent 执行时强制遵守，软偏好可被 AI 优化、硬规则必须被 AI 服从；
  - **永久记忆**：偏好 / 规则 / 历史行程持久化存储、跨会话生效，从"一次性工具"进化为"长期私人规划助手"——你三年前说过怕坐长途车，它现在还记得。

**为什么这条路线重要**

这不是一个一次性 demo 的功能堆叠，而是一条清晰的技术演进路线——从"AI 出方案"进化到"AI 协作规划"。它契合 TRAE"AI 协作开发"的核心理念，也让我们对 agent 架构的边界与可能性有了更深的思考：**真正的智能不是一次答对，而是在多轮交互中逐步逼近最优解**。
