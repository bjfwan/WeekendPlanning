## 截图 01 · 首页需求表单（Hero + 一键填充示例）

- **拍什么画面**
  首页完整态：顶部"3 分钟 · AI 生成"徽章 + 主标题"周末去哪玩？3 分钟帮你搞定" + 数据证明条（3 min / 4 类人群 / 1 份清单）；下方表单卡片中"基础信息 / 出行配置 / 个性偏好"三组已填好示例值（杭州、明日、一日游、预算 500、2 人、公共交通、偏好「放松·美食·自然风光」）；底部"一键填充示例"与"生成周末行程"按钮可见。
- **突出亮点**
  产品价值：3 分钟从需求到完整路线的极简体验；交互细节：时长/出行方式图标卡片选中态、日期快捷选项（今天/明天/本周末）、表单分组分隔线滚动绘制动画。
- **建议操作步骤**
  1. 打开体验地址进入首页；
  2. 点击中下方的「一键填充示例」按钮（ghost 样式），表单自动填入杭州示例；
  3. 等待 Hero 入场动画与数字滚动结束，全屏截图（建议桌面端 1440 宽）。
- **建议图注文案**
  「输入城市、心情、预算和人数，一键填充示例即可 3 秒进入生成环节——AI 周末规划师首页表单。」

## 截图 02 · AI 流式生成 · 4 阶段状态机（核心，强烈推荐）

- **拍什么画面**
  生成页（PlanView）的 StreamingPanel：顶部 4 阶段轨道「理解需求 → 生成方案 → 优化细节 → 完成」，其中阶段 2「生成方案」为高亮激活态（coral 红色 + pulse 脉冲），连接线呈流动光效；下方显示"🧠 AI 正在思考最佳方案..."，思考过程区域实时滚动输出推理文本与闪烁光标；可同时截到"正在补充地图坐标... 3 / 5"进度条。
- **突出亮点**
  技术亮点：**SSE 流式 + 4 阶段状态机**，用真实阶段轨道替代假进度条，进度反映真实生成状态（reasoning/content 长度驱动阶段切换）；坐标补充进度条体现后端逐点反查高德地理编码的过程。
- **建议操作步骤**
  1. 在首页填好示例后点击「生成周末行程」跳转生成页；
  2. 待页面出现"思考中"与阶段轨道后，等待约 3–5 秒，让阶段 2 激活、连接线流动光效出现；
  3. 在思考过程文字滚动 + 光标闪烁瞬间全屏截图。
- **建议图注文案**
  「不是假进度条！4 阶段状态机实时反映 AI 生成进度——理解需求 → 生成方案 → 优化细节 → 完成，SSE 流式逐字输出推理过程。」

## 截图 03 · 高德地图路线绘制结果（核心，强烈推荐）

- **拍什么画面**
  生成完成后的结果页「当日路线地图」区域：高德地图上依次绘制 5 个行程标记点（灵隐寺 → 断桥 → 三潭印月 → 雷峰塔 → 河坊街），点间连线为按地理最近邻优化的路线；右上角「🗺️ 按地理 / ⏰ 按时间」切换器可见，当前选中"按地理"。
- **突出亮点**
  技术亮点：**高德地图路线绘制 + 地理最近邻重排**（routeOptimizer），后端按地理顺序重排点位减少折返；地图组件 defineAsyncComponent 懒加载（首屏不加载地图 SDK，对应 vendor 分包优化）。
- **建议操作步骤**
  1. 生成完成（或点击「🎯 加载演示行程」立即进入结果页）；
  2. 滚动到「当日路线地图」区域，等待地图标记与连线绘制完成；
  3. 全屏截图，确保 5 个标记点与连线、右上角排序切换器均可见。
- **建议图注文案**
  「高德地图自动绘制当日路线，5 个景点按地理最近邻重排、减少折返——地图组件按需懒加载，首屏 bundle 从 128KB 降到 6KB。」

## 截图 04 · 时间轴 + 行程卡片联动

- **拍什么画面**
  结果页时间轴区域：纵向时间轴竖线（coral→amber→mint 渐变）串联 5 个行程卡片，每个卡片含时间（08:30–10:30）、标题（灵隐寺）、描述、花费、交通、Tips；首个节点为 coral、末节点为 mint；底部"第 1 天合计 ¥250"徽章。可截一张"点击地图标记后时间轴卡片高亮（coral 描边 + scale 放大）"的联动状态。
- **突出亮点**
  产品价值：结构化时间轴 + 地图标记点击联动（点击地图标记自动滚动定位到对应卡片并高亮 2 秒）；GSAP stagger 入场与节点逐个 pulse 动效。
- **建议操作步骤**
  1. 在结果页滚动到时间轴区域，等待入场动画（竖线绘制 + 卡片 stagger）完成；
  2. （可选联动）点击地图上的某个标记，让对应卡片高亮后截图；
  3. 全屏截图，包含时间轴竖线、节点、至少 2–3 张行程卡片与底部合计徽章。
- **建议图注文案**
  「纵向时间轴串联全天行程，点击地图标记自动定位并高亮卡片——结构化呈现 + 地图/时间轴双向联动。」

## 截图 05 · 多人协同分享弹窗 + 加入页

- **拍什么画面**
  左侧/主屏：结果页点击「🔗 分享链接」后弹出的分享弹窗——mint 绿色成功图标（带脉冲环扩散）、"分享链接已生成"、行程信息卡片、分享链接输入框、"复制链接 / 打开链接"按钮；右侧/第二窗口：用另一个浏览器（无痕窗口）打开分享链接后的「加入行程协同」页——顶部"多人协同规划"徽章、分享码、行程基本信息卡、加入表单（昵称 + 偏好标签 +「✨ 加入协同」按钮）。
- **突出亮点**
  技术亮点：**匿名登录 + Supabase Realtime**，无需注册即可生成分享链接；成员通过链接加入后偏好实时同步给创建者，创建者可"基于成员偏好重新生成"行程。
- **建议操作步骤**
  1. 在结果页点击「🔗 分享链接」，弹窗出现后截图（主窗口）；
  2. 点击「打开链接」在无痕窗口打开加入页，填入昵称与偏好后截图（第二窗口）；
  3. 两张图并排呈现"分享—加入"闭环，或分别提交。
- **建议图注文案**
  「一键生成分享链接，小伙伴匿名加入并提交偏好——Supabase Realtime 实时同步成员，创建者可基于全员偏好重新生成行程。」

  3. 全屏截图，确保城市徽章、标题、总花费、天气条均可见。
- **建议图注文案**
  「总花费数字从 0 弹性滚动到 250，天气智能提示伴随预算对照——结果呈现也有仪式感。」

## 截图 07 · 演示兜底 + 超时保护（容错体验）

- **拍什么画面**
  方案 A（演示兜底）：生成失败/超时后页面底部出现"或先体验演示行程效果" +「🎯 加载演示行程」按钮，点击后立即展示完整杭州一日游结果（地图/时间轴/清单齐全）。方案 B（超时提示）：生成超过 30 秒时 StreamingPanel 底部弹出黄色超时提示条"生成时间较长，是否继续等待？"+「继续等待 / 取消」按钮。
- **突出亮点**
  技术亮点：**50s 超时保护 + 演示兜底**，AI 生成失败/超时不阻断评委体验，可一键加载真实坐标的演示行程；前端 30s 友好提示 + 后端 50s 硬超时双重保护。
- **建议操作步骤**
  1. （演示兜底）若现场生成失败，点击「🎯 加载演示行程」后截结果页；
  2. （超时提示）若生成耗时较长，等待 30 秒让超时提示条出现后截图；
  3. 优先截"演示兜底"画面，体现容错与评委友好。
- **建议图注文案**
  「AI 偶尔抽风也不怕——50s 超时保护 + 一键加载演示行程，评委体验零阻断。」

## 截图 08 · Trae IDE 开发过程（核心，强烈推荐）

- **拍什么画面**
  Trae IDE 工作区截图，展示本项目的真实开发场景，任选其一：
  - **A. 4 阶段状态机代码**：打开 `src/components/StreamingPanel.vue`，展示 `stages` computed（理解需求/生成方案/优化细节/完成）与阶段切换逻辑；
  - **B. SSE 流式解析**：打开 `src/composables/useStream.ts`，展示 `parseSSEBuffer` 与 `meta/reasoning/chunk/progress/done` 事件分发；
  - **C. vendor 分包配置**：打开 `vite.config.ts`（或 Trae 终端 `pnpm build` 输出），展示 manualChunks 把高德/Supabase 等拆为独立 vendor chunk，首屏 chunk 从 128KB 降到 6KB；
  - **D. JWT 修复**：打开 `server/src/middleware/auth.ts`，展示匿名会话 JWT 校验修复点。
- **突出亮点**
  工程亮点：用 Trae IDE 完成 SSE 流式、4 阶段状态机、vendor 分包、JWT 修复、超时保护等关键工程实现，体现 AI 辅助研发的完整链路。
- **建议操作步骤**
  1. 在 Trae IDE 中打开本项目仓库；
  2. 优先选 A（状态机）或 C（分包构建产物），最能直观体现工程深度；
  3. 截图包含左侧文件树 + 中间代码 + Trae 标识（如窗口标题/侧边栏），让评委一眼识别"在 Trae 里开发"。
- **建议图注文案**
  「在 Trae IDE 中完成 SSE 流式、4 阶段状态机、vendor 分包（首屏 128KB→6KB）、JWT 修复等关键工程——AI 辅助研发全链路。」
